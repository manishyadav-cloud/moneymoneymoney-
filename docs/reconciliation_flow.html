e.g. custGen_8700600197_195885339_1
Juspay juspay_txn_id = 'wiom-' + order_id + '-1' = PhonePe Merchant Order Id = PayU txnid = Paytm Order_ID (after TRIM quotes)
e.g. wiom-custGen_8700600197_195885339_1-1

⚠ Key data issues:
1. Paytm IDs have embedded single quotes — all joins require TRIM(col, chr(39))
2. Razorpay joins on order_id = order_receipt (not juspay_txn_id like other PGs) — no separate settlement table
3. Bank receipts are daily aggregates (358 rows) vs 1M+ individual settlements — reconciliation is amount-level, not txn-level
4. Wiom non-online (cash) transactions in primary_revenue/net_income do NOT appear in Juspay — this is expected
5. Juspay has 1.1M rows vs 20K booking rows — booking only captures a subset; net_income (647K) and primary_revenue (391K) cover much more
